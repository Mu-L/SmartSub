# 日志中心

任务失败、服务连不上、下载超时——第一步都是看日志。日志中心汇集应用运行日志与系统消息，用于排查问题与提交反馈。

## 打开方式

- 顶栏**帮助菜单 → 查看日志**
- 应用菜单里的「查看日志」
- 设置页「关于」卡片中的入口

## 筛选查询

- **按日期**：日志按天存储（JSONL 文件），下拉切换任意保留期内的日期
- **按条数**：默认显示最新 100 条，可调整
- **按类型**：信息 / 警告 / 错误等级过滤，快速定位异常

警告与错误会高亮显示。翻译术语命中、引擎回退等关键行为也会写入日志，方便核查。

## 复制与清理

- **复制日志**：一键复制当前查询结果，[提 Issue](https://github.com/buxuku/SmartSub/issues) 时贴上它能大幅加快定位
- **清空日志**：手动清空
- **自动清理**：日志保留 **7 天**（含当天），过期文件启动时自动删除，不会无限占用磁盘

## 反馈问题的建议

提交 Issue 时附上：

1. 日志中心复制的相关日志（错误发生时间前后）
2. 应用版本（状态栏右下角）与操作系统
3. 复现步骤与所用引擎 / 服务商
