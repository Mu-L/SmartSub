# 自定义 OpenAI 兼容服务

只要对方提供 **OpenAI 风格 `/v1` 协议**的接口，就能接进妙幕当翻译服务——中转站、自建 vLLM / LM Studio、国产大模型平台（混元、Kimi、智谱等）全都适用，并且**可以添加多个实例**分别管理。

## 在妙幕中配置

「翻译」页面点侧栏顶部「**+**」添加自定义服务商，选择 OpenAI API 模板：

| 字段     | 填写                                                                                                                           |
| -------- | ------------------------------------------------------------------------------------------------------------------------------ |
| API 地址 | 服务的基础地址，**通常以 `/v1` 结尾**（如 `https://api.openai.com/v1`）。不要粘贴 `/chat/completions` 完整路径——应用会自动拼接 |
| API Key  | 对方签发的密钥（本地自建服务随意填一个占位）                                                                                   |
| 模型名称 | 服务商提供的**准确模型 ID**（如 `hunyuan-3.0-preview`、`moonshot-v1-8k`）                                                      |

点「**测试翻译**」验证——测试会自动探测该端点支持的**结构化输出模式**并写回配置。

## 典型接法

- **API 中转站**：Base url 填中转地址 + 中转 Key，模型名照中转站文档
- **LM Studio（本地）**：`http://localhost:1234/v1`，Key 任意，模型名为已加载模型
- **vLLM（自建）**：`http://<服务器>:8000/v1`
- **国产平台**：各家的 OpenAI 兼容端点均可（注意部分平台的兼容端点是独立域名）

## 常见问题

- **测试提示 response_format 不支持**：把该服务的「结构化输出」改为 Disabled（配置页高级选项）
- **404**：Base url 多写或少写了路径段——确认以 `/v1` 结尾
- **译文错位**：开启「回显对齐校验」
