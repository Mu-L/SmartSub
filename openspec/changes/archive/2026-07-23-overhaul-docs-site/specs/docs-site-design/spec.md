## ADDED Requirements

### Requirement: 落地页重设计传达当前产品形态

落地页 SHALL 重写为以下区块序列：Hero（价值主张 + 版本号下载按钮 + 产品截图）、流水线区（转写 → 翻译 → 校对 → 配音 → 合成）、数字条（引擎数/翻译服务数/音色数/平台数/免费）、分功能图文区块（对应核心功能篇并链接）、全流程免费方案区、CTA。数字类表述 MUST 与 v3.5.0 事实一致。

#### Scenario: 首屏传达完整价值

- **WHEN** 用户首次访问站点首页
- **THEN** 首屏可见一句话价值主张（覆盖字幕生成、翻译、配音、合成）、当前版本号的下载入口与 v3.5 新 UI 产品截图

#### Scenario: 功能区块可跳转

- **WHEN** 用户点击落地页任一功能区块的链接
- **THEN** 跳转到对应核心功能文档篇

### Requirement: 全站视觉系统统一且专业

站点 SHALL 在 `custom.css` 建立统一的 design tokens（品牌色、圆角、阴影、间距、中文字体栈），正文排版 MUST 优化阅读体验（合理的行宽上限、行高、标题层级），并保持 Docusaurus classic 框架、不引入额外样式框架。

#### Scenario: 视觉风格一致

- **WHEN** 用户在落地页、文档页、更新日志页之间切换
- **THEN** 配色、圆角、卡片、按钮风格一致，无 Docusaurus 默认绿色主题残留

#### Scenario: 不新增样式依赖

- **WHEN** 检查 `docs/package.json` 依赖变更
- **THEN** 无 Tailwind 等新样式框架依赖

### Requirement: 深浅色模式完整支持

落地页与全部自定义组件 SHALL 在深色与浅色模式下均正常显示，文本对比度 MUST 满足可读性（无深色模式下的低对比或反白问题）。

#### Scenario: 深色模式无样式破损

- **WHEN** 用户切换到深色模式并浏览落地页与文档页
- **THEN** 所有区块背景、文字、卡片、截图边框正常显示，无不可读文本

### Requirement: 移动端响应式可用

落地页与文档页 SHALL 在移动端视口（≤768px）正常布局：Hero 纵向堆叠、数字条换行、图文区块单列、表格可横向滚动。

#### Scenario: 移动端浏览落地页

- **WHEN** 用户以 375px 宽度视口访问首页
- **THEN** 无横向溢出，全部区块纵向排列且可读
