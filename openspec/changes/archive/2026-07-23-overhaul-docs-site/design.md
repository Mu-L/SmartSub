# Design: 文档站全面翻新

## Context

文档站位于 `docs/`，Docusaurus 3 classic preset，`routeBasePath: '/'`（文档即站点主体）+ `src/pages/index.tsx` 落地页，仅 zh-Hans，部署于 Vercel（smartsub.buxuku.com）。内容最后一次系统性更新在 v2.3 时代；软件当前版本 v3.5.0。

可用素材：

- 根 `README.md`：已重写至 v3.4 口径（引擎对比表、8 家云端听写配置说明、免费方案表、GPU 加速矩阵），是内容重写的主要蓝本；v3.5 的视频下载/术语表需从 release note 补充。
- `Changelog/v3.2.0–v3.5.0`：中英双语 release note；v3.0/v3.1 需从 GitHub Releases 提取。
- `resources/preview/`：home/merge/profread 新 UI 截图（zh/en 各一套）。
- `types/provider.ts`：20 个翻译服务商的字段级配置事实（凭据字段、默认值）。
- 应用页面清单（`renderer/pages/[locale]/`）：home、tasks、download、dubbing、engines、glossary、proofread、subtitleMerge、translateControl、ttsServices、settings 等，即截图覆盖面。

## Goals / Non-Goals

**Goals:**

- 文档内容与 v3.5.0 代码事实一致，无 2.x 时代残留描述。
- 服务商配置指南做到「小白照着做就能配通」：每服务商独立页、统一模板、真实截图。
- 落地页传达产品当前形态（五段流水线、免费可跑通），视觉达到与产品 UI 匹配的专业度。
- 高频搜索场景（视频加字幕/转文字/AI 配音等）有直接命中的着陆页。
- 构建通过 `onBrokenLinks: 'throw'`，Vercel 可直接部署。

**Non-Goals:**

- 不做英文版内容（仅保留 Docusaurus i18n 标准目录约定，后续可增 `docs/i18n/en/`）。
- 不改应用代码；不重构 `docs/ui-v3-proposal/`、`docs/smart-ui/`（不在路由内，不影响构建）。
- 不做换主题/swizzle 级深度定制，不引入 Tailwind 等新样式框架。
- 不做更新日志自动同步脚本（手工维护，成本可接受）。
- 不做竞品对比页（避免口碑风险，SEO 只做需求词不做竞品词）。

## Decisions

### D1 信息架构：按流水线组织功能篇，配置指南按目录自动生成侧边栏

新侧边栏结构（`sidebars.ts`）：

```
入门指南        introduction / installation / quickstart
核心功能        subtitle-generation / subtitle-translation / proofread /
               tts-dubbing / video-merge / video-download
配置指南        guides/engines/（选型总览 + 7 引擎页 + 云端听写 8 家）
               guides/translation/（选型总览 + 20 服务商页）
               guides/tts/（选型总览 + 本地 3 引擎页 + 云端 5 服务商页）
场景教程        scenarios/（4 篇起步）
进阶使用        gpu-acceleration / glossary / prompts / recipes / storage / logs
FAQ · 更新日志 · 下载 · 开发说明
```

配置指南三个目录用 `_category_.json` + autogenerated 子项，避免 30+ 页面在 `sidebars.ts` 里手工罗列；排序用 frontmatter `sidebar_position`（推荐/免费的排前）。

**备选**：沿用现有「模型与配置」大页 + 锚点——被否，用户已决策按服务商独立成页（SEO 长尾 + 移动端阅读体验）。

### D2 服务商页统一模板 + 轻量 MDX 元信息卡

每页固定结构：

```markdown
---
title: <服务商> 配置指南 # 即搜索着陆词
description: <一句话：是什么 + 免费额度/计费>
keywords: [SmartSub, <服务商>, 字幕翻译/语音识别/配音, 配置]
sidebar_position: N
---

<ProviderMeta> 官网 | 凭据类型 | 免费额度 | 计费方式 | 推荐场景 </ProviderMeta>

## 适用场景与费用

## 申请步骤（编号列表 + 官方控制台截图可选）

## 在妙幕中配置（应用内截图，必有）

## 参数说明（字段对照 types/provider.ts 核实）

## 常见问题与报错

> 信息更新于 2026-07，费用政策以官方页面为准
```

`ProviderMeta` 为 `docs/src/components/` 下的简单展示组件（props 传值，非数据驱动）。内容手写，不从 `provider.ts` 生成——申请步骤/费用本来就无法生成，数据驱动是过度工程；但字段名必须对照代码核实。

### D3 截图体系：运行应用 + CDP 驱动截屏，深色主题单套（实施中修订）

- `yarn dev` 启动应用（临时环境变量开启 CDP 远程调试端口，截图后还原代码），CDP `Page.captureScreenshot` 截取视口（1280×872 逻辑分辨率 / Retina 2x），zh 界面。
- **深色主题单套**（修订：原定浅色）：应用默认即深色，README 既有官方宣传图全为深色，统一深色保证品牌视觉一致。
- 每张图落到 `docs/static/img/v3/<area>/<name>.webp`（cwebp q90，1440px 宽），旧 `screenshots/`、`features/` 目录整体删除。
- 演示数据不跑真实转写链：校对批次经 `createProofreadTask` IPC 构造（30s 合成语音样例视频 + 手写 SRT），配音/合成页用 `?subtitle=&video=` 查询参数直载。
- 凭据安全：密码框原生打码；明文输入框（AppID 等）由截图脚本按启发式规则加 `webkit-text-security` 视觉遮罩（不改动 React 状态），全部截图逐张人工复核。

### D4 落地页：重写 index.tsx 区块，纯 Infima 变量建立视觉系统

- 组件结构：`Hero`（左文案+下载按钮带 `customFields.appVersion`，右产品截图）→ `PipelineStrip`（转写→翻译→校对→配音→合成 五段横条）→ `StatsBar`（7 类引擎 / 20 翻译服务 / 300+ 音色 / 3 平台 / 全流程免费）→ `FeatureSections`（5–6 个图文交替区块，对应核心功能篇并链接过去）→ `FreePathSection`（免费方案表，核心差异点）→ `FAQ 精选` → `CTA`。
- 视觉系统在 `custom.css` 建立 design tokens：品牌主色沿用应用 UI 色系、圆角/阴影/间距变量、中文字体栈（system-ui 优先，不引入 webfont 避免拖慢加载）、代码块与 admonition 微调、正文 `max-width` 与行高优化阅读体验。
- 深浅色模式全覆盖（Infima `[data-theme='dark']` 变量层）。
- 不引入组件库/Tailwind：Docusaurus 内 Infima + CSS Modules 足够，避免升级负担。

### D5 SEO：frontmatter 元数据为主 + 全站修复项

- 每篇文档 frontmatter 必填 `title`/`description`/`keywords`；场景教程页 H1 直接使用搜索需求词（如「YouTube 视频怎么加中文字幕」）。
- `docusaurus.config.ts`：补 `themeConfig.metadata`（keywords/og:type 等）、修复 OG 图（制作 `static/img/smartsub-social-card.png` 1200×630，替换失效引用）、`title` 调整为「妙幕 SmartSub - 视频字幕生成 · 翻译 · AI 配音一站式工具」形态。
- 首页经 `<Head>` 注入 JSON-LD `SoftwareApplication`（含 operatingSystem/offers free）。
- sitemap 由 preset-classic 自带（production 构建生成），确认未被禁用即可；`onBrokenLinks: 'throw'` 保持。
- 关键词分层：核心词（视频字幕生成、字幕翻译、AI 配音、声音克隆、视频转文字、开源免费）落在首页与功能篇；长尾词（服务商名 + 配置/免费额度）天然落在配置指南页；场景词落在场景教程页。

### D6 更新日志与内容清理

- `changelog.md` 重写：v3.5 → v3.0 倒序，每版取 release note 中文亮点 3–6 条 + GitHub Release 链接；2.x 折叠为一段历史概述。
- 删除：`tutorial-basics/`、`tutorial-extras/`、`blog/`（config 已 `blog: false`）、`static/img/undraw_*.svg`、`docusaurus.png`、旧截图目录；`docs/docs/tasks/` 三篇孤儿页内容并入对应功能篇后删除，如有外链价值则在新页保留同名锚点。
- `development.md`/`download.md` 核对更新（构建命令 `yarn native:fetch`、Homebrew 安装、GPU 加速包应用内下载等新事实）。

## Risks / Trade-offs

- [截图依赖本地环境：模型未下载、演示数据缺失导致空态截图] → 截图前准备清单：预下载 1 个小模型（tiny/base）、准备 30s 样例视频跑通任务链；云端配置页允许用「已配置未连接」状态截图。
- [服务商申请流程/费用信息随时间失效] → 每页脚注「信息更新于 YYYY-MM」+ 官方文档链接兜底；申请步骤只写稳定主干（注册→开通→取凭据），控制台细节截图从简。
- [30+ 新页面广撒网，部分服务商页内容单薄] → 允许小众服务商页短小（Meta 卡 + 配置截图 + 字段说明即可），质量线是「照着能配通」，不强求篇幅。
- [手工维护的功能清单/数字条（20 个翻译服务等）随版本漂移] → 数字类表述集中在首页与介绍页两处，release 流程中随 README 一并核对（在 tasks 中留核对项）。
- [落地页重写引入构建回归] → 保持 classic preset API，`yarn build` 作为验收门槛，Vercel 预览环境人工过一遍。
- [Edge TTS 属逆向接口、DeepLX 稳定性差，指南若过度承诺易引发负面反馈] → 此类服务商页明确标注「不承诺可用性/试用档」，与 README 口径一致。
